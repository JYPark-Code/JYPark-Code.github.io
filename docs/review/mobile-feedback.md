# 모바일 화면 리뷰 및 수정 기준

> 상태: 리뷰 완료 · 작성일 2026-07-11 · 기준 커밋 `d12241f`  
> 대상: GitHub Pages 정적 export 포트폴리오 홈

## 1. 리뷰 목적

현재 디자인의 데스크톱 인상은 유지하면서, 모바일 방문자가 첫 화면에서 핵심 정보와 주요 조작 요소를 놓치지 않도록 한다.

이 문서는 구현 코드를 지정하는 설계서가 아니라 다음 수정 작업의 판단 기준이다. 권장안과 다른 방식으로 구현해도 아래 완료 조건을 만족하면 된다.

## 2. 확인 환경

- `npm run build`로 생성한 `out/` 정적 산출물
- Microsoft Edge(Chromium) 헤드리스 렌더링
- 모바일 뷰포트: **390 × 844px**
- 데스크톱 비교 뷰포트: **1440 × 1200px**
- 기본 상태: 영문 / 다크 테마
- 품질 게이트: `typecheck`, `lint`, `build` 모두 통과

## 4. 상세 피드백

### 4.1 P0 — 상단 언어·테마 버튼이 화면 밖으로 밀림

#### 관찰 결과

390px 너비에서 브랜드와 `CONTACT` 링크까지는 보이지만, 그 뒤에 배치된 `KO`와 테마 버튼이 오른쪽 화면 밖으로 밀린다. 모바일 사용자는 언어와 테마를 전환할 수 없다.

#### 관련 코드

- `src/components/Nav.tsx`: `contact`, 방문자 수, 환경설정 버튼이 하나의 `.nav-links` 행에 배치됨
- `src/app/globals.css`: 700px 이하에서 `.sec` 링크만 숨김
- `.nav-inner`의 좌우 패딩, `.nav-links a` 패딩, `.icon-btn`의 마진이 좁은 화면에서도 유지됨

#### 원인 후보

모바일 미디어 쿼리가 `work`, `journey`, `about`만 숨기고, 남은 요소들의 간격과 패딩은 데스크톱 값을 그대로 사용한다. 브랜드와 오른쪽 컨트롤의 합산 너비가 가용 폭을 초과한다.

#### 권장 수정 방향

- 700px 이하에서 `contact` 텍스트 링크도 숨기거나 별도 모바일 메뉴로 이동한다.
- 또는 520px 이하에서 내비게이션 좌우 패딩과 버튼 간격을 줄인다.
- 언어·테마 버튼은 모바일에서도 항상 노출되는 핵심 컨트롤로 취급한다.
- 방문자 수는 현재처럼 900px 이하에서 숨겨도 된다.

#### 완료 조건

- [ ] 320px, 360px, 390px, 430px 너비에서 가로 오버플로가 없다.
- [ ] 브랜드, 언어 버튼, 테마 버튼이 동시에 보인다.
- [ ] 언어·테마 버튼을 터치해도 인접 버튼이 잘못 눌리지 않는다.
- [ ] 200% 브라우저 확대에서도 핵심 컨트롤이 화면 밖으로 밀리지 않는다.

### 4.2 P1 — 통계 카드가 모바일에서 지나치게 크게 쌓임

#### 관찰 결과

520px 이하에서 히어로와 통계 카드가 단일 열로 변환된다. 첫 번째 통계 카드가 히어로에 가까운 큰 높이로 표시되고, 같은 형태의 카드 3개가 이어지므로 프로젝트 목록이 첫 화면에서 멀어진다.

#### 관련 코드

- `.hero-bento`: `grid-auto-rows: 1fr`
- 520px 이하: `grid-template-columns: 1fr`
- `.stat`: 별도 모바일 높이 또는 패딩 축소 규칙 없음

#### 권장 수정 방향

가장 권장하는 구성은 모바일에서 **히어로 1행 + 통계 3열의 압축 행**이다.

```text
┌─────────────────────────┐
│          Hero           │
└─────────────────────────┘
┌───────┬───────┬─────────┐
│ Full  │Java/TS│Web·Infra│
└───────┴───────┴─────────┘
```

- 모바일 구간에서 `grid-auto-rows`를 `auto`로 재설정한다.
- 히어로는 전체 열을 사용하고 통계 3개는 같은 행에 배치한다.
- 통계 값의 글자 크기와 카드 패딩을 모바일에 맞게 낮춘다.
- 320px에서 3열 가독성이 부족하면 `2열 + 마지막 카드 전체 폭` 구성을 대안으로 사용한다.

#### 완료 조건

- [ ] 세 통계 카드가 히어로와 같은 높이로 늘어나지 않는다.
- [ ] 390 × 844px에서 대표 프로젝트 섹션 제목이 약 1.5화면 이내에 나타난다.
- [ ] `Full-stack`, `Java/TS`, `Web·Infra`가 잘리거나 겹치지 않는다.
- [ ] 한글 전환 후에도 라벨이 카드 경계를 넘지 않는다.

### 4.3 P1 — 첫 화면의 정보 밀도와 프로젝트 진입 속도

#### 관찰 결과

히어로 자체의 시각적 완성도와 CTA는 좋다. 다만 모바일에서는 히어로, 큰 통계 카드 3개, 섹션 여백이 순서대로 쌓여 포트폴리오의 핵심인 프로젝트 확인이 늦어진다.

#### 권장 수정 방향

- 히어로의 핵심 카피와 두 CTA는 유지한다.
- 통계 영역을 압축한 뒤 히어로와 `Browse` 사이의 모바일 세로 여백을 다시 조정한다.
- 프로젝트 카드 너비는 다음 카드의 일부가 보이게 유지해 가로 스와이프 가능성을 알린다.
- 모바일 첫 진입에서는 장식 요소보다 `Browse work` CTA와 대표 프로젝트 노출을 우선한다.

#### 완료 조건

- [ ] 첫 화면에서 이름, 역할, 한 문단 설명, `Browse work`, GitHub CTA가 모두 읽힌다.
- [ ] 첫 번째 CTA를 누르면 sticky nav에 섹션 제목이 가려지지 않는다.
- [ ] 한 번 또는 두 번의 일반적인 세로 스와이프로 대표 프로젝트에 도달한다.
- [ ] 가로 프로젝트 목록에서 첫 카드와 다음 카드의 진입 단서가 보인다.

### 4.4 P2 — 터치 타깃 크기

#### 관찰 결과

`.icon-btn`은 높이 2rem, 프로젝트·타임라인 이동 버튼도 대체로 2rem이다. 데스크톱 마우스에는 충분하지만 모바일 터치에서는 조작 여유가 작다.

#### 권장 수정 방향

- 보이는 원의 크기를 반드시 키울 필요는 없지만 클릭 가능한 영역은 최소 **44 × 44px**에 가깝게 확보한다.
- 버튼 사이에 충분한 간격을 둔다.
- hover에만 의존하지 않고 `:active`와 `:focus-visible` 상태를 유지한다.

#### 완료 조건

- [ ] 상단 언어·테마 버튼의 터치 영역이 각각 충분히 분리된다.
- [ ] 프로젝트/타임라인 좌우 버튼을 한 손 엄지로 반복 조작할 수 있다.
- [ ] 키보드 포커스 표시가 모바일 에뮬레이션에서도 잘리지 않는다.

### 4.5 P2 — 모바일 모달과 포커스 흐름

#### 코드 검토 결과

모달은 640px 이하에서 한 열로 전환되고 Escape 닫기를 지원한다. 그러나 열릴 때 모달로 포커스를 이동하거나, 포커스를 모달 내부에 가두거나, 닫은 뒤 원래 프로젝트 카드로 돌려보내는 처리는 없다.

이 문제는 모바일 화면 배치 자체보다는 모바일 키보드, 스위치 컨트롤, 보조 기술 사용 시 영향을 준다.

#### 권장 수정 방향

- 모달이 열리면 닫기 버튼 또는 모달 패널로 포커스를 이동한다.
- Tab/Shift+Tab 포커스가 열린 모달 안에서 순환하도록 한다.
- 닫을 때 모달을 연 프로젝트 카드로 포커스를 복귀시킨다.
- 모바일에서 긴 모달을 끝까지 스크롤한 뒤에도 닫기 수단을 쉽게 찾을 수 있게 한다.

#### 완료 조건

- [ ] 키보드만으로 카드 열기 → 내용 탐색 → 닫기 → 기존 카드 복귀가 가능하다.
- [ ] 모달이 열린 동안 뒤쪽 페이지 요소로 포커스가 이동하지 않는다.
- [ ] 320px 너비에서 제목, 본문, 메타데이터, 링크가 가로로 잘리지 않는다.
- [ ] iOS Safari와 Android Chrome에서 배경 스크롤이 잠긴다.

## 6. 권장 작업 순서

1. 내비게이션 오버플로 수정
2. 모바일 통계 레이아웃 압축
3. 히어로와 프로젝트 사이의 세로 간격 조정
4. 터치 타깃 확대
5. 모달 포커스 처리
6. 위 뷰포트·상태 조합으로 브라우저 회귀 확인
7. `npm run typecheck` → `npm run lint` → `npm run build`
